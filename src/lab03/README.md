# Постановка задачи

На основе существующего класса из ЛР-1 необходимо построить **иерархию классов**

Пример:

```
Product
 ├── FoodProduct
 ├── DigitalProduct
 └── Service
```

```
Student
 ├── BachelorStudent
 ├── MasterStudent
 └── PhDStudent
```

```
Book
 ├── PrintedBook
 ├── Ebook
 └── AudioBook
```

---

# Реализованная иерархия

Должно быть:

* 1 базовый класс (из ЛР-1)
* минимум 2 производных класса

---

# Задание на 3

## Требования

1. Использовать **класс из ЛР-1 как базовый**.

2. Создать минимум **2 дочерних класса**.

3. Добавить в дочерние классы:
   * 2 новых атрибута
   * минимум 1 новый метод

4. Использовать `super()` для вызова конструктора базового класса.

5. Не дублировать код базового класса.

---

## Демонстрация

В `demo.py`:

* создание объектов разных типов
* вывод объектов
* использование методов базового и дочернего класса

---

# Задание на 4

## Требования

0. Реализовать всё из задания на 3

1. Переопределить метод базового класса:

    Например:

    ```
    __str__()
    ```

    или бизнес-метод

2. Использовать:

    ```
    isinstance()
    ```

    для проверки типов

3. Добавить **полиморфное поведение**

    Например:

    ```
    calculate_price()
    ```

    который работает по-разному в разных классах

4. Интеграция с коллекцией (ЛР-2):

* коллекция должна уметь хранить **разные типы объектов(само собой из наследованных от базового)**
* работать с ними корректно

---

## Демонстрация

В `demo.py`:

* работа с разными типами через одну коллекцию
* вызов одного метода — разное поведение
* проверка типов через `isinstance()`

---

# Задание на 5

## Требования

0. Реализовать всё из заданий на 3 и 4

1. Ввести **общий интерфейс поведения через базовый класс**

    Вводим простой интерфейс использования общего метода, например:
    ```
    process()
    calculate()
    display()
    ```

2. Реализовать **полиморфизм без условий**

    Анти-паттерн:

    ```
    if type == ...
    ```

    Гуд-паттерн:

    ```
    obj.calculate()
    ```

3. Добавить взаимодействие с коллекцией:

    * фильтрация по типу
    * выборка только одного вида объектов

        Например:

        ```
        get_only_digital()
        get_only_active()
        ```

---

## Демонстрация

В `demo.py`:

* единый список объектов разных типов
* вызов одинакового метода для разных типов и получение разных результатов
* фильтрация по типу
* минимум 3 сценария
