    Аннотировать:
    * параметры конструктора
    * возвращаемые значения методов
    * атрибуты в `__init__`

    Пример:

    ```python
    class Student:
        def __init__(self, name: str, gpa: float, year: int) -> None:
            self._name: str = name
            self._gpa: float = gpa

        def get_name(self) -> str:
            return self._name
    ```

2. Создать в `lab06/container.py` класс **`TypedCollection`** — Generic-версию коллекции из ЛР-2.

    Коллекция из ЛР-2 оставить нетронутой. `TypedCollection` — это новый класс, который повторяет её интерфейс, но теперь знает какой тип внутри хранится.

    ```python
    from typing import TypeVar, Generic

    T = TypeVar('T')

    class TypedCollection(Generic[T]):
        def __init__(self) -> None:
            self._items: list[T] = []

        def add(self, item: T) -> None:
            self._items.append(item)

        def remove(self, item: T) -> None:
            self._items.remove(item)

        def get_all(self) -> list[T]:
            return list(self._items)
    ```

    **!! Все методы которые были в коллекции из ЛР-2 — перенести сюда с правильными аннотациями типов !!**

# Задание на 4

## Требования

0. Реализовать всё из задания на 3.

1. Добавить в `TypedCollection` три новых метода с корректными аннотациями типов.

    **`find`** — принимает функцию-условие, возвращает первый подходящий элемент или `None`:

    ```python
    from typing import Callable, Optional

    def find(self, predicate: Callable[[T], bool]) -> Optional[T]:
        for item in self._items:
            if predicate(item):
                return item
        return None
    ```

    **`filter`** — принимает функцию-условие, возвращает список всех подходящих элементов:

    ```python
    def filter(self, predicate: Callable[[T], bool]) -> list[T]:
        return [item for item in self._items if predicate(item)]
    ```

    **`map`** — принимает функцию-преобразование, применяет её к каждому элементу и возвращает список результатов. Тип результата может быть другим — для этого нужен второй TypeVar `R`:

    ```python
    R = TypeVar('R')

    def map(self, transform: Callable[[T], R]) -> list[R]:
        return [transform(item) for item in self._items]
    ```

2. Продемонстрировать что `map` реально меняет тип результата.

    Например: была `TypedCollection[Student]`, применили разные функции — получили `list[str]` (имена) и `list[float]` (GPA). Это наглядно показывает зачем нужен второй TypeVar.

## Демонстрация

В `demo.py` показать:

* вызов `find()` — один раз элемент найден, один раз не найден (результат `None`)
* вызов `filter()` — показать отфильтрованный список
* вызов `map()` — два раза с разными функциями, чтобы было видно что тип результата меняется

# Задание на 5

## Требования

0. Реализовать всё из заданий на 3 и 4.

1. Реализовать **два Protocol** в файле `container.py`.

    Protocol описывает что должен уметь объект — без наследования, просто по наличию нужных методов:

    ```python
    from typing import Protocol

    class Displayable(Protocol):
        def display(self) -> str:
            ...

    class Scorable(Protocol):
        def score(self) -> float:
            ...
    ```

    Классы из ЛР-1 и ЛР-3 **не должны явно наследоваться** от этих протоколов — достаточно что у них есть соответствующие методы. Если методов нет — добавить.

2. Создать **TypeVar с ограничением** (`bound=`) на каждый Protocol.

    ```python
    D = TypeVar('D', bound=Displayable)
    S = TypeVar('S', bound=Scorable)
    ```

    Теперь `TypedCollection[D]` — это не коллекция чего угодно, а только тех объектов у которых есть метод `display()`. Внутри коллекции можно смело вызывать `item.display()` — IDE знает что это безопасно.

3. Продемонстрировать что объекты разных типов из иерархии ЛР-3 подходят под Protocol **без явного наследования**.

    ```python
    # PrintedBook и AudioBook не наследуются от Displayable —
    # но у обоих есть метод display(), значит оба подходят
    books: TypedCollection[D] = TypedCollection()
    books.add(PrintedBook(...))
    books.add(AudioBook(...))
    ```

## Демонстрация

В `demo.py` — минимум 2 сценария:

* **Сценарий 1:** `TypedCollection[D]` с объектами разных типов из иерархии ЛР-3 — показать что объекты попадают в коллекцию без наследования от Protocol, и что методы протокола вызываются корректно для каждого типа
* **Сценарий 2:** `TypedCollection[S]` — то же самое, но через второй Protocol. Показать что один и тот же класс `TypedCollection` работает с разными ограничениями
