# Задание на 3

## Требования

1. Реализовать минимум **3 функции-стратегии для сортировки** объектов коллекции.

    Примеры стратегий сортировки:
    * по имени / заголовку / названию
    * по числовому атрибуту (цена, рейтинг, год, возраст)
    * по нескольким атрибутам одновременно

2. Использовать функции сортировки с параметром `key=`:

    ```python
    sorted(collection, key=by_price)
    collection.sort(key=lambda x: x.name)
    ```

3. Реализовать минимум **2 функции-фильтра**.

    Примеры:
    * фильтрация по условию на атрибут (цена < X, рейтинг > Y)
    * фильтрация по типу объекта (с использованием `isinstance`)

4. Применить встроенную функцию `filter()`:

    ```python
    result = list(filter(is_expensive, products))
    ```

5. Передавать функции как аргументы (передавать **ссылку**, а не вызывать функцию внутри).

## Демонстрация

В `demo.py` должно быть показано:

* создание коллекции объектов (минимум 5 штук)
* сортировка одной коллекции **тремя разными стратегиями** — вывод каждый раз
* фильтрация коллекции двумя разными функциями-фильтрами

# Задание на 4

## Требования

0. Реализовать всё из задания на 3.

1. Применить `map()` для преобразования коллекции.

    Примеры:
    * преобразование объектов в строки/словари
    * применение скидки ко всем товарам
    * извлечение одного поля из всех объектов

    ```python
    names = list(map(lambda x: x.name, collection))
    discounted = list(map(apply_discount(0.1), products))
    ```

2. Реализовать **фабрику функций** — функцию, которая создаёт и возвращает другую функцию.

    ```python
    def make_price_filter(max_price):
        def filter_fn(item):
            return item.price <= max_price
        return filter_fn
    ```

3. Добавить методы `sort_by(key_func)` и `filter_by(predicate)` в класс коллекции из ЛР-2.

    ```python
    my_collection.sort_by(by_price)
    my_collection.filter_by(is_available)
    ```

4. Использовать `lambda` там, где функция проста и не переиспользуется.

## Демонстрация

* применение `map()` — показать результат преобразования
* использование фабрики функций — создать фильтр с параметром и применить
* вызов методов `sort_by()` / `filter_by()` коллекции
* сравнение: один и тот же результат через `lambda` и через именованную функцию

# Задание на 5

## Требования

0. Реализовать всё из заданий на 3 и 4.

1. Реализовать **паттерн «Стратегия»** через `callable`-объекты.

    Создать несколько стратегий обработки объектов коллекции. Стратегия должна быть взаимозаменяемой без изменения кода коллекции:

    ```python
    class DiscountStrategy:
        def __call__(self, item):
            return item.price * 0.9
    ```

2. Реализовать метод `apply(func)` в коллекции — применить произвольную функцию ко всем элементам.

    ```python
    collection.apply(activate)
    collection.apply(lambda x: x.upgrade())
    ```

3. Реализовать **цепочку операций** над коллекцией.

    Операции должны выполняться последовательно, каждая принимает результат предыдущей:

    ```python
    result = (collection
        .filter_by(is_available)
        .sort_by(by_price)
        .apply(apply_discount))
    ```

4. Вынести все стратегии и функции-обработчики в отдельный файл `strategies.py`. Функции должны быть задокументированы (`docstring`).

## Демонстрация

В `demo.py` — минимум 3 сценария:

* **Сценарий 1:** полная цепочка `filter → sort → apply` с выводом на каждом шаге
* **Сценарий 2:** замена стратегии без изменения кода коллекции — показать, что при передаче другой функции результат меняется
* **Сценарий 3:** демонстрация `callable`-объекта как стратегии
