# Требования к реализации

## Задание на 3

### Требования

1. Создать **минимум 2 интерфейса (ABC)**.

2. В каждом интерфейсе:

    * минимум 1 абстрактный метод (`@abstractmethod`)

3. Реализовать интерфейсы в существующих классах:

    * минимум 2 класса должны реализовывать интерфейс

4. Методы интерфейса должны иметь **разную реализацию** в разных классах.

---

### Пример

```
Printable → to_string()
Comparable → compare_to(other)
```

---

### Демонстрация

В `demo.py`:

* создание объектов разных типов
* вызов интерфейсных методов
* разное поведение у разных классов

---

## Задание на 4

### Требования

0. Реализовать всё из задания на 3

1. Использовать интерфейс **как тип**

Пример:

```python
def print_all(items: list[Printable]):
    for item in items:
        print(item.to_string())
```

2. Добавить **универсальную функцию**, работающую через интерфейс

3. Добавить второй интерфейс в те же или другие классы

(множественная реализация интерфейсов)

4. Проверка через `isinstance()`

    ```python
    isinstance(obj, Printable)
    ```

---

### Демонстрация

* работа функции, работающей с разными объектами через интерфейс
* использование `isinstance`
* показать, что объекты реализуют несколько интерфейсов

---

## Задание на 5

### Требования

0. Реализовать всё из заданий на 3 и 4

1. Интеграция с коллекцией (ЛР-2):

Коллекция должна уметь:
 - работать с объектами через интерфейс
 - фильтровать по интерфейсу

Пример:
```python
get_printable()
get_comparable()
```

---

2. Полиморфизм через интерфейс (без условий)

Плохо:

```python
if isinstance(obj, A):
```

Хорошо:

```python
obj.to_string()
```

---

3. Добавить **архитектурное поведение**

Например:

* сортировка через `Comparable`
* вывод через `Printable`

---

### Демонстрация

В `demo.py`:

* единый список объектов разных типов
* работа через интерфейсы
* фильтрация коллекции по интерфейсу
* минимум 3 сценария
