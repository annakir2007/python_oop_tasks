# Задание на 3

## Требования

1. Реализовать **консольное меню** с минимум 5 пунктами.

   Пример для предметной области «Книга»:
   ```
   1. Добавить книгу
   2. Показать все книги
   3. Найти книгу
   4. Удалить книгу
   0. Выход
   ```

2. Каждый пункт меню вызывает соответствующую операцию над коллекцией.

3. Приложение должно работать в **цикле**: после каждого действия снова показывать меню.

4. Обработка **некорректного ввода**: неверный пункт меню, ввод строки вместо числа и т.д.

   ```python
   try:
       choice = int(input("Выберите пункт: "))
   except ValueError:
       print("Ошибка: введите число")
   ```

5. Разделить код на минимум **2 файла**:
   * `main.py` — точка входа
   * `cli.py` или аналог — логика меню

## Демонстрация

Скриншоты интерактивной работы в терминале:
* добавление нескольких объектов
* вывод коллекции
* попытка ввода некорректных данных и обработка ошибки
* выход из программы

# Задание на 4

## Требования

0. Реализовать всё из задания на 3.

1. Реализовать **собственные исключения** для ошибок предметной области.

   ```python
   class ItemNotFoundError(Exception):
       """Объект не найден в коллекции."""
       pass

   class DuplicateItemError(Exception):
       """Объект с таким идентификатором уже существует."""
       pass
   ```

2. Вынести исключения в отдельный файл `exceptions.py`.

3. Добавить в меню операции **поиска и фильтрации** (из ЛР5):
   * поиск по атрибуту
   * фильтрация по условию (например, по диапазону цен или статусу)

4. Реализовать **разделение на 3 слоя**:
   * `cli.py` — только ввод/вывод, никакой бизнес-логики
   * `app.py` — бизнес-логика, операции над коллекцией
   * `models.py` — классы предметной области

   CLI не должен напрямую обращаться к коллекции — только через `app.py`.

5. Улучшить вывод: форматированные таблицы или структурированный вывод объектов.

## Демонстрация

* работа поиска и фильтрации через меню
* демонстрация перехвата собственного исключения (например, попытка добавить дубликат)
* видно, что вывод аккуратно отформатирован

# Задание на 5

## Требования

0. Реализовать всё из заданий на 3 и 4.

1. Реализовать **сохранение и загрузку данных** через файл (JSON или CSV).

   ```python
   # storage.py
   def save(collection, filepath: str) -> None:
       """Сохранить коллекцию в JSON-файл."""
       ...

   def load(filepath: str) -> list:
       """Загрузить объекты из JSON-файла."""
       ...
   ```

   При запуске приложение автоматически загружает данные; при выходе — сохраняет.

2. Добавить в меню **сортировку** (из ЛР5) с выбором стратегии:

   ```
   Сортировать по:
   1. Названию
   2. Цене
   3. Дате добавления
   ```

3. Подключить **типизацию** (из ЛР6): все методы в `app.py` и `cli.py` должны иметь аннотации типов.

4. Все публичные функции и методы должны иметь **docstring**.

5. Реализовать **подтверждение опасных операций**:

   ```
   Удалить "Мастер и Маргарита"? (y/n):
   ```

## Демонстрация

В скриншотах показать минимум 4 сценария:

* **Сценарий 1:** запуск → автозагрузка данных из файла → вывод коллекции
* **Сценарий 2:** добавление, редактирование или удаление с подтверждением → выход → повторный запуск (данные сохранились)
* **Сценарий 3:** сортировка с выбором стратегии через меню
* **Сценарий 4:** фильтрация + перехват исключения при некорректных данных
